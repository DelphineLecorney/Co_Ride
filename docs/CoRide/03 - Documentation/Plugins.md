# <span style="color:#00C896;">Plugins Obsidian utilisés</span>


Ce dossier documente les plugins utilisés pour organiser, visualiser et enrichir la documentation du projet **Co_Ride**.
%% ## <span style="color:#3A7AFE;">Plugins principaux</span> %%

---
### <span style="color:#FF8A00;"><strong>**Dataview**</strong></span>
> [!info] Rôle : créer des vues dynamiques (tables, listes, requêtes) sur les fichiers Markdown.


> [!success] Utilisation : index des domaines, listes automatiques, tableaux de suivi.
> - Intérêt : navigation intelligente dans la documentation.

---

### <span style="color:#FF8A00;"><strong> **Excalidraw**</strong></span>

> [!info] Rôle : création de schémas, diagrammes et croquis.


> [!success] Utilisation : architecture, flux métier, modèles DDD.
> - Intérêt : visuels intégrés directement dans le vault.

---



### <span style="color:#FF8A00;"><strong> **Templater**</strong></span>
> [!info] Rôle : Automatisation de modèles Markdown.


> [!success]  Utilisation : templates pour Domain.md, Aggregates.md, Events.md, UseCases.md.
> - Intérêt : cohérence documentaire.

---

> [!warning] **Note importante**  
> - Ces plugins ne font pas partie du code Co_Ride. 
> - Ils servent uniquement à **structurer et enrichir la documentation**.
